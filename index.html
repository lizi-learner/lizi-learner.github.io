面向对象与原型

发表于 2021-06-08 更新于 2021-06-09

创建对象

工厂模式

这种工厂模式可以解决创建多个类似对象的问题，但没有解决对象标识问题，即所有对象都是Object类

构造函数模式

要创建Person的实例，应使用new 操作符，以这种方式调用构造函数会执行如下操作：

  • 在内存中创建一个对象
  • 这个新对象内部的[[Prototype]]特性被赋值为构造函数的prototype属性
  • 构造函数内部的this被赋值为这个新对象
  • 执行构造函数内的代码
  • 如果构造函数返回非空对象，则返回该对象；否则返回刚创建的新对象

构造函数的问题

构造函数的主要问题在于，其定义的方法会在每个实例上都创建一遍。要解决这个问题，可以把函数定义转移到构造函数外部，但这会扰乱全局作用域。因此，原型模式来创建对象是最好的。

原型模式

理解原型

无论何时，只要创建一个函数，就会按照特定的规则执行：

  • 为这个函数创建一个prototype属性，指向原型对象
  • 原型对象自动获得一个名为constructor的属性，指向构造函数

在调用构造函数创建新实例时：

  • 实例的内部[[Prototype]]特性被赋值为构造函数的原型对象
  • 实例对象上会暴露__proto__属性（[[Prototype]]特性的getter和setter方法），可以访问原型
  • 最好不要使用__proto__属性，而是使用Object.getPrototypeOf()和Object.setPrototypeOf()实现读写

原型层级

在通过对象访问属性时，会按照这个属性的名称开始搜索。搜索开始于对象实例本身，如果没有找到这个属性，则搜索会沿着指针进入原型对象，然后在原型对象中寻找。这就是原型用于在多个对象实例间共享属性和方法的原理。

原型和in操作符

  • in操作符会在可以通过对象访问指定属性时返回true，无论属性在原型还是实例上
  • for…in会遍历可枚举，可通过对象访问的非符号属性

继承

继承是通过原型实现的。有以下几种方式实现继承

  • 原型链
  • 盗用构造函数
  • 组合继承
  • 寄生式组合继承
  • 类（ES6语法糖）

这里只记录原型链，寄生式组合继承和类

原型链

原型链的基本思想就是通过原型继承多个引用类型的属性和方法。

创建原型链的最佳方案就是一个对象的原型是另一个对象的实例。

原型链：teacher -> new Person() -> new Object()

默认原型

任何函数的默认原型都是一个Object的实例。

注意这还有一点：任意内建构造函数的原型(__proto)是Function的原型，包括Function

原型和继承关系

原型和实例的关系可以通过三种方式来确定

  • instanceof 操作符

    • 语法：obj instanceof Class

    • 执行过程：

      • 如果Class有静态方法Symbol.hasInstance，那么直接调用这个方法

      • 如果没有该方法，js引擎检查Class的原型是否存在于obj的原型链上

  • isPrototypeOf()

    • 语法：objA.isPrototypeOf(objB)
    • 如果 objA处在 objB 的原型链中，则返回 true

  • Object.prototype.toString 方法

  • 内建对象类型

    • 对于 number 类型，结果是 [object Number]
    • 对于 boolean 类型，结果是 [object Boolean]
    • 对于 null：[object Null]
    • 对于 undefined：[object Undefined]
    • 对于数组：[object Array]

  • 自定义对象类型

    可以使用特殊的对象属性 Symbol.toStringTag 自定义对象的 toString 方法的行为。

  • 总结

    用于 返回值
    typeof 原始数据类型 string
    {}.toString 原始数据类型，内建对象，包含 Symbol.toStringTag 属性的对象 string
    instanceof 对象 true/false

原型链的问题

  • 原型中包含的引用值会在所有实例间共享
  • 子类型在实例化时不能给父类型的构造函数传参
  • constructor属性被覆盖。在继承之前,Teacher.prototype.constructor等于Teacher，使用原型链后，Teacher.prototype.constructor等于Person

寄生式组合继承

类

类定义

  • 类声明：class Person {}
  • 类表达式：const Animal = class {};

类可以包含构造函数方法，实例方法，getter/setter方法，静态类方法。这些都不是必需的。

类表达式的名称是可选的。在把类表达式赋给变量后，可以通过name属性取得类表达式的名称字符串，但不能在类表达式外的作用域外部访问这个标识符

类构造函数

  • 类构造函数用constructor关键字定义，会告诉解释器在使用new 操作符创建类的新实例时，应该调用这个函数。
  • 不定义构造函数相当于构造函数是空函数。

实例化

  • 使用new操作符实例化类相当于使用new调用类的构造函数，执行的操作和直接使用构造函数创建实例是一样的
  • 类实例化传入的参数会用作构造函数的参数
  • 类构造函数和构造函数的区别：调用类构造函数必须使用new 操作符

类是一种特殊函数

  • typeof操作符检测类标识符，表明类是一个函数

  • 类有prototype属性，而这个原型也有一个constructor属性指向类

  • 可以用instanceof检查构造函数原型是否存在于实例的原型链中

  • 类中定义的constructor方法不会被当成构造函数，在对它使用instanceof操作符时会返回false。但如果在创建实例时直接将类构造函数当成普通构造函数来使用，那么instanceof操作符返回值会反转。

  • 类是JavaScript的一等公民，可以像其他对象和函数引用一样把类作为参数传递

实例、原型和类成员

  • 类构造函数内部定义的属性叫做实例成员，所有实例成员不会在实例之间共享

  • 类块中定义的实例方法是原型方法

    在类块中使用箭头函数定义的方法是实例成员而不是原型方法

  • 静态类方法是类的属性

继承

虽然类继承使用的是新语法，但背后依旧使用的是原型链。

重写方法

子类通常希望在父类方法的基础上进行调整或扩展其功能。此时可以使用super关键字

  • 执行super.method(…)来调用一个父类方法（实例方法和静态方法）
  • 执行super(…)来调用一个父类constructor（只能在子类constructor中调用）

重写constructor

  • 调用super()会调用父类构造函数，并将返回的实例赋值给this

  • super()的行为如同调用构造函数，如果需要给父类构造函数传参，则需要手动传入

  • 如果没有定义类构造函数，在实例化子类时会调用super()，而且会传入所有传给子类的参数

  • 在类构造函数中，不能在调用super()之前引用this

  • 如果在子类中显式定义了构造函数，则要么必须在其中调用super()，要么必须在其中返回一个对象

    在 JavaScript 中，继承类的构造函数与其他函数之间是有区别的。

    派生构造器具有特殊的内部属性 [[ConstructorKind]]:"derived"。这是一个特殊的内部标签。

    该标签会影响它的 new 行为：

    • 当通过 new 执行一个常规函数时，它将创建一个空对象，并将这个空对象赋值给 this。
    • 但是当继承的 constructor 执行时，它不会执行此操作。它期望父类的 constructor 来完成这项工作。

    因此，派生的 constructor 必须调用 super 才能执行其父类（base）的 constructor，否则 this 指向的那个对象将不会被创建。并且我们会收到一个报错。

[[HomeObject]]

为了提供解决方法，JavaScript 为函数添加了一个特殊的内部属性：[[HomeObject]]。

  • 方法在内部的 [[HomeObject]] 属性中记住了它们的类/对象。这就是 super 如何解析父方法的。
  • 因此，将一个带有 super 的方法从一个对象复制到另一个对象是不安全的。

正如我们之前所知道的，函数通常都是“自由”的，并没有绑定到 JavaScript 中的对象。正因如此，它们可以在对象之间复制，并用另外一个 this 调用它。

[[HomeObject]] 的存在违反了这个原则，因为方法记住了它们的对象。[[HomeObject]] 不能被更改，所以这个绑定是永久的。

在 JavaScript 语言中 [[HomeObject]] 仅被用于 super。所以，如果一个方法不使用 super，那么我们仍然可以视它为自由的并且可在对象之间复制。但是用了 super 再这样做可能就会出错。

[[HomeObject]] 是为类和普通对象中的方法定义的。但是对于对象而言，方法必须确切指定为 method()，而不是 "method: function()"。

这个差别对我们来说可能不重要，但是对 JavaScript 来说却非常重要。

遍历对象属性

发表于 2021-06-07

javascript遍历对象的属性有多种不同的方法，主要原因有以下几点

  • 对象属性有可枚举和不可枚举之分，这是由属性的enumerable描述符决定的（包括数据属性和访问器属性）
  • 对象可能会从原型对象上继承属性
  • 对象属性有常规属性（数据属性和访问器属性）和符号属性(Symbol)

不同的遍历方法会遍历不同范围的属性。

  • for in 循环

    for…in循环会遍历可枚举的自身属性和继承属性（可枚举，继承的常规属性，即不包含符号属性）

    如果想用for…in并想排除继承的属性，可以使用内建方法obj.hasOwnProperty(key)：如果 obj具有自己的（非继承的）名为 key的属性，则返回 true。

  • Object.keys() / Object.values()

    遍历自身可枚举的属性（可枚举，非继承属性，不包含符号属性）

  • Object.getOwnPropertyNames()方法

    返回对象实例的常规属性数组，不包含符号属性（可枚举，不可枚举，非继承，非符号）

  • Object.getOwnPropertyDescriptors()方法

    返回同时包含常规和符号属性描述的对象（非继承）

  • Object.getOwnPropertySymbols()方法

    返回对象实例的符号属性数组（非继承）

函数调用

发表于 2021-06-07

函数有两个隐式的函数参数，this和arguments。两者会被静默的传递给函数，并且可以向函数体内显式声明的参数意义被正常访问。这两个参数在Javascript代码中至关重要。由于调用函数的方式对函数的隐式参数有很大影响，我们需要理解函数的调用。

隐含参数

  • arguments

    arguments参数是一个类数组结构，能够允许我们访问传递给函数的所有参数

  • this

    this 代表函数调用相关联的对象，通常称为函数上下文。

    this参数的指向不仅是由定义函数的方式和位置决定的，同时还严重受到函数调用方式的影响。

函数调用

可以通过4种方式调用一个函数：

  • 作为一个函数(function)，直接被调用：example()
  • 作为一个方法(method)，关联在一个对象上被调用：obj.example()
  • 作为一个构造函数(constructor)，被调用：new Example()
  • 通过函数的apply或call方法被调用：example.call(obj, …args) 或 example.apply(obj, args)

  • 非严格模式下，它将是全局上下文对象（浏览器中是window对象，nodejs中是global对象）
  • 严格模式下，this等于undefined

作为方法被调用

当一个函数被赋值给一个对象的属性，并且通过对象属性引用的方式调用函数时，函数会作为对象的方法被调用。

此时this的值是引用该方法的对象

作为构造函数被调用

使用关键字new会触发以下几个动作：

  • 创建一个新的空对象
  • 该对象作为this参数传递给构造函数
  • 执行构造函数内的代码
  • 新构造的对象作为new运算符的返回值

当函数被作为构造函数被调用时，有两点需要注意：

  • 构造函数的返回值：如果构造函数显式返回一个对象，则该对象将作为整个表达式的值返回，而传入的this将被丢弃，但如果构造函数显式返回的是非对象类型，则忽略返回值，返回this.
  • 构造函数也可以作为一个简单函数被直接调用，但最好不要这样做。

使用apply和call方法调用

不同于函数调用时默认将函数上下文传入作为this参数，apply方法和call方法运行调用一个显式设置this的函数。

语法如下

  • func.call(context, arg1, arg2, …)：context作为this，后面的是函数的参数
  • func.apply(context, args)：context作为this，args是函数的参数，只接受类数组对象（有length属性, 其它属性（索引）为非负整数）

解决函数上下文问题

当将对象方法作为回调进行传递，例如传递给setTimeout，可能会出现函数上下文与预期不符合的问题。

解决方案

结合以上函数调用知识，有如下几种解决方案

  • 使用包装函数
  • 使用call或apply显示传递this
  • 使用箭头函数绕过this的默认传递
  • 绑定函数：bind()方法
    • 语法：let bound = func.bind(context, [arg1], [arg2], ...),
    • 它允许将上下文绑定为 this，以及绑定函数的起始参数。
    • 注意
      • 绑定函数的上下文是硬绑定（hard-fixed）的。没有办法再修改它。
      • 一个函数不能被重绑定（re-bound）
      • bind 的返回结果是另一个对象

闭包和作用域

发表于 2021-06-05 更新于 2021-06-06

闭包

当在外部函数中声明内部函数时，不仅定义了函数的声明，而且还创建了一个闭包。该闭包不仅包含了函数的声明，还包含了在函数声明时该作用域中的所有变量。当最终执行内部函数时，尽管声明的作用域已经消失了，但是通过闭包，仍然能够访问到原始作用域。

执行上下文

通过执行上下文跟踪代码。

执行上下文

JS引擎执行代码时，每一条语句都处于特定的执行上下文中。

有两种执行上下文：

  • 全局执行上下文：当JS程序开始执行时就创建了全局上下文
  • 函数执行上下文：在每次调用函数时，就会创建一个新的函数执行上下文

JS是单线程，在某个特定的时刻只能执行特定的代码，一旦发生函数调用，当前的执行上下文必须停止执行，并创建新的函数执行上下文来执行函数。当函数执行完成后，将函数执行上下文销毁，并重新回到发生调用时的执行上下文中。所以需要跟踪执行上下文。

执行上下文栈

执行上下文栈又称为调用栈。跟踪执行上下文，即正在执行的上下文和正在等待的上下文，最简单的方法是使用调用栈。

词法环境

词法环境是js引擎内部用来跟踪标识符与特定变量之间的映射关系。通常称为作用域。

在JS中，每个运行的函数、代码块{...}以及整个脚本，都有一个被称为词法环境的内部的关联对象。

词法环境由两部分组成：

  • 环境记录：存储所有局部变量作为其属性的对象
  • 对外部词法环境的引用。

以下演示了代码执行过程中全局词法环境的变化：

  • 当脚本开始运行，词法环境预先填充了所有声明的变量，他们处于undefined状态
  • 但函数声明会立即变为即用型函数

访问变量

当代码要访问一个变量时 —— 首先会搜索内部词法环境，然后搜索外部环境，然后搜索更外部的环境，以此类推，直到全局词法环境。

如果在任何地方都找不到这个变量，那么在严格模式下就会报错（在非严格模式下，为了向下兼容，给未定义的变量赋值会创建一个全局变量）。

闭包原理

所有函数都有名为 [[Environment]] 的隐藏属性，该属性保存了对创建该函数的词法环境的引用。

因此，在函数内声明的函数在[[Environment]]保存了外部函数的词法环境（即外部作用域）的引用，因此就算执行内部函数时，外部作用域已经消失了，就比如内部函数被外部函数返回并在外部执行时（如下），内部函数还是可以访问到外部函数的词法环境。

闭包的缺点

通常，函数调用完成后，会将词法环境和其中的所有变量从内存中删除。因为现在没有任何对它们的引用了。与 JavaScript 中的任何其他对象一样，词法环境仅在可达时才会被保留在内存中。

但是，如果有一个嵌套的函数在函数结束后仍可达，则它将具有引用词法环境的 [[Environment]] 属性。即使在函数执行完成后，词法环境仍然可达。

obj.keys

  • Set
  • Array

Object.keys，values，entries

对于普通对象可以使用以下方法迭代

  • Object.values(obj)：返回一个包含该对象所有的值的数组。
  • Object.entries(obj)：返回一个包含该对象所有 [key, value] 键值对的数组。

可以利用以上方法让对象能够使用数组方法

  • Object.fromEntries(array)：将数组转换为对象

  • 转换对象

    对象缺少数组存在的许多方法，例如 map 和 filter 等。

    如果我们想应用它们，那么我们可以使用 Object.entries，然后使用 Object.fromEntries：

    • 使用 Object.entries(obj) 从 obj 获取由键/值对组成的数组。

    • 对该数组使用数组方法，例如 map。

    • 对结果数组使用 Object.fromEntries(array) 方法，将结果转回成对象。

不同的相等比较方式

发表于 2021-06-03

Javascript中有多种不同的比较方式，我们需要在不同的情况采用不同的方式做相等比较。

相等比较算法

  • The Abstract Equality Comparison Algorithm (==)
  • The Strict Equality Comparison Algorithm (===)
  • SameValue (Object.is())
  • SameValueZero (暂未提供API)

==

等于操作符会先进行强制类型转换然后再确定操作数是否相等。遵循以下规则

  • 不同类型的值在比较时会被转换为数值再比较，false转换为0，true转换为1
  • null和undefined相等，它们不能转换为其他类型的值进行比较(仅限于相等操作符)
  • NaN与任何值都不相等，包括它自己。

===

严格相等运算符 === 在进行比较时不会做任何的类型转换。

  • 只要操作数类型不同就返回false
  • null不等于undefined
  • NaN不等于NaN

SameValue

SameValueZero

在map，set，includes方法中使用的比较算法是SameValueZero，它与sameValue的区别只有0等于-0
