函数调用

发表于 2021-06-07

函数有两个隐式的函数参数，this和arguments。两者会被静默的传递给函数，并且可以向函数体内显式声明的参数意义被正常访问。这两个参数在Javascript代码中至关重要。由于调用函数的方式对函数的隐式参数有很大影响，我们需要理解函数的调用。

隐含参数

  • arguments

    arguments参数是一个类数组结构，能够允许我们访问传递给函数的所有参数

  • this

    this 代表函数调用相关联的对象，通常称为函数上下文。

    this参数的指向不仅是由定义函数的方式和位置决定的，同时还严重受到函数调用方式的影响。

函数调用

可以通过4种方式调用一个函数：

  • 作为一个函数(function)，直接被调用：example()
  • 作为一个方法(method)，关联在一个对象上被调用：obj.example()
  • 作为一个构造函数(constructor)，被调用：new Example()
  • 通过函数的apply或call方法被调用：example.call(obj, …args) 或 example.apply(obj, args)

作为方法被调用

当一个函数被赋值给一个对象的属性，并且通过对象属性引用的方式调用函数时，函数会作为对象的方法被调用。

此时this的值是引用该方法的对象

作为构造函数被调用

使用关键字new会触发以下几个动作：

  • 创建一个新的空对象
  • 该对象作为this参数传递给构造函数
  • 执行构造函数内的代码
  • 新构造的对象作为new运算符的返回值

当函数被作为构造函数被调用时，有两点需要注意：

  • 构造函数的返回值：如果构造函数显式返回一个对象，则该对象将作为整个表达式的值返回，而传入的this将被丢弃，但如果构造函数显式返回的是非对象类型，则忽略返回值，返回this.
  • 构造函数也可以作为一个简单函数被直接调用，但最好不要这样做。

使用apply和call方法调用

不同于函数调用时默认将函数上下文传入作为this参数，apply方法和call方法运行调用一个显式设置this的函数。

语法如下

  • func.call(context, arg1, arg2, …)：context作为this，后面的是函数的参数
  • func.apply(context, args)：context作为this，args是函数的参数，只接受类数组对象（有length属性, 其它属性（索引）为非负整数）

解决函数上下文问题

当将对象方法作为回调进行传递，例如传递给setTimeout，可能会出现函数上下文与预期不符合的问题。

解决方案

结合以上函数调用知识，有如下几种解决方案

  • 使用包装函数
  • 使用call或apply显示传递this
  • 使用箭头函数绕过this的默认传递
  • 绑定函数：bind()方法
    • 语法：let bound = func.bind(context, [arg1], [arg2], ...),
    • 它允许将上下文绑定为 this，以及绑定函数的起始参数。
    • 注意
      • 绑定函数的上下文是硬绑定（hard-fixed）的。没有办法再修改它。
      • 一个函数不能被重绑定（re-bound）
      • bind 的返回结果是另一个对象

箭头函数

再重点记录一下箭头函数

箭头函数不仅仅是编写简洁代码的“捷径”。它还具有非常特殊且有用的特性。

JavaScript 充满了我们需要编写在其他地方执行的小函数的情况。

例如：

  • arr.forEach(func) —— forEach 对每个数组元素都执行 func。
  • setTimeout(func) —— func 由内建调度器执行。

在这样的函数中，我们通常不想离开当前上下文

没有this

箭头函数没有 this。如果访问 this，则会从外部获取。

但箭头函数就没事，因为它们没有 this。

不能使用new

不具有 this 自然也就意味着另一个限制：箭头函数不能用作构造器（constructor）。不能用 new 调用它们。

没有arguments

箭头函数也没有 arguments 变量。

当我们需要使用当前的 this 和 arguments 转发一个调用时，这对装饰器（decorators）来说非常有用。

箭头函数vsbind

箭头函数 => 和使用 .bind(this) 调用的常规函数之间有细微的差别：

  • .bind(this) 创建了一个该函数的“绑定版本”。
  • 箭头函数 => 没有创建任何绑定。箭头函数只是没有 this。this 的查找与常规变量的搜索方式完全相同：在外部词法环境中查找。

总结

箭头函数：

  • 没有 this
  • 没有 arguments
  • 不能使用 new 进行调用
  • 它们也没有 super
