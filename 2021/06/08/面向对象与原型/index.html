面向对象与原型

发表于 2021-06-08

创建对象

工厂模式

这种工厂模式可以解决创建多个类似对象的问题，但没有解决对象标识问题，即所有对象都是Object类

构造函数模式

要创建Person的实例，应使用new 操作符，以这种方式调用构造函数会执行如下操作：

  • 在内存中创建一个对象
  • 这个新对象内部的[[Prototype]]特性被赋值为构造函数的prototype属性
  • 构造函数内部的this被赋值为这个新对象
  • 执行构造函数内的代码
  • 如果构造函数返回非空对象，则返回该对象；否则返回刚创建的新对象

构造函数的问题

构造函数的主要问题在于，其定义的方法会在每个实例上都创建一遍。要解决这个问题，可以把函数定义转移到构造函数外部，但这会扰乱全局作用域。因此，原型模式来创建对象是最好的。

原型模式

理解原型

无论何时，只要创建一个函数，就会按照特定的规则执行：

  • 为这个函数创建一个prototype属性，指向原型对象
  • 原型对象自动获得一个名为constructor的属性，指向构造函数

在调用构造函数创建新实例时：

  • 实例的内部[[Prototype]]特性被赋值为构造函数的原型对象
  • 实例对象上会暴露__proto__属性（[[Prototype]]特性的getter和setter方法），可以访问原型
  • 最好不要使用__proto__属性，而是使用Object.getPrototypeOf()和Object.setPrototypeOf()实现读写

原型层级

在通过对象访问属性时，会按照这个属性的名称开始搜索。搜索开始于对象实例本身，如果没有找到这个属性，则搜索会沿着指针进入原型对象，然后在原型对象中寻找。这就是原型用于在多个对象实例间共享属性和方法的原理。

例：虽然rabbit实例对象没有constructor属性，但会访问到原型对象上的constructor属性

obj.hasOwnProperty()方法用于确定某个属性在实例上还是在原型对象上。

原型和in操作符

  • in操作符会在可以通过对象访问指定属性时返回true，无论属性在原型还是实例上
  • for…in会遍历可枚举，可通过对象访问的非符号属性

继承

继承是通过原型实现的。有以下几种方式实现继承

  • 原型链
  • 盗用构造函数
  • 组合继承
  • 寄生式组合继承
  • 类（ES6语法糖）

这里只记录原型链，寄生式组合继承和类

原型链

原型链的基本思想就是通过原型继承多个引用类型的属性和方法。

创建原型链的最佳方案就是一个对象的原型是另一个对象的实例。

原型链：teacher -> new Person() -> new Object()

默认原型

任何函数的默认原型都是一个Object的实例

原型和继承关系

原型和实例的关系可以通过两种方式来确定

  • instanceof操作符

    teacher instanceof Person：js引擎检查Person的原型是否存在于teacher的原型链上

  • isPrototypeOf()

原型链的问题

  • 原型中包含的引用值会在所有实例间共享
  • 子类型在实例化时不能给父类型的构造函数传参
  • constructor属性被覆盖。在继承之前,Teacher.prototype.constructor等于Teacher，使用原型链后，Teacher.prototype.constructor等于Person

类

类定义

  • 类声明：class Person {}
  • 类表达式：const Animal = class {};

类可以包含构造函数方法，实例方法，getter/setter方法，静态类方法。这些都不是必需的。

类表达式的名称是可选的。在把类表达式赋给变量后，可以通过name属性取得类表达式的名称字符串，但不能在类表达式外的作用域外部访问这个标识符

类构造函数

  • 类构造函数用constructor关键字定义，会告诉解释器在使用new 操作符创建类的新实例时，应该调用这个函数。
  • 不定义构造函数相当于构造函数是空函数。

实例化

  • 使用new操作符实例化类相当于使用new调用类的构造函数，执行的操作和直接使用构造函数创建实例是一样的
  • 类实例化传入的参数会用作构造函数的参数
  • 类构造函数和构造函数的区别：调用类构造函数必须使用new 操作符

类是一种特殊函数

  • typeof操作符检测类标识符，表明类是一个函数

  • 类有prototype属性，而这个原型也有一个constructor属性指向类

  • 可以用instanceof检查构造函数原型是否存在于实例的原型链中

  • 类中定义的constructor方法不会被当成构造函数，在对它使用instanceof操作符时会返回false。但如果在创建实例时直接将类构造函数当成普通构造函数来使用，那么instanceof操作符返回值会反转。

  • 类是JavaScript的一等公民，可以像其他对象和函数引用一样把类作为参数传递

实例、原型和类成员

  • 类构造函数内部定义的属性叫做实例成员，所有实例成员不会在实例之间共享

  • 类块中定义的实例方法是原型方法

    在类块中使用箭头函数定义的方法是实例成员而不是原型方法

  • 静态类方法是类的属性

继承

虽然类继承使用的是新语法，但背后依旧使用的是原型链。

继承基础

使用extends关键字，就可以继承任何拥有[[constructor]]和原型的对象

  • 继承类

  • 继承普通构造函数

在内部，关键字extends使用了旧的原型机制进行工作，他将Student.prototype.[[Prototype]]设置为Person.prototype（换一种说法，把Student.prototype，设置为原型等于Person.prototype的对象），相当于实现了原型链。

重写方法

子类通常希望在父类方法的基础上进行调整或扩展其功能。此时可以使用super关键字

  • 执行super.method(…)来调用一个父类方法（实例方法和静态方法）
  • 执行super(…)来调用一个父类constructor（只能在子类constructor中调用）

重写constructor

  • 调用super()会调用父类构造函数，并将返回的实例赋值给this

  • super()的行为如同调用构造函数，如果需要给父类构造函数传参，则需要手动传入

  • 如果没有定义类构造函数，在实例化子类时会调用super()，而且会传入所有传给子类的参数

  • 在类构造函数中，不能在调用super()之前引用this

  • 如果在子类中显式定义了构造函数，则要么必须在其中调用super()，要么必须在其中返回一个对象

    在 JavaScript 中，继承类（所谓的“派生构造器”，英文为 “derived constructor”）的构造函数与其他函数之间是有区别的。派生构造器具有特殊的内部属性 [[ConstructorKind]]:"derived"。这是一个特殊的内部标签。

    该标签会影响它的 new 行为：

    • 当通过 new 执行一个常规函数时，它将创建一个空对象，并将这个空对象赋值给 this。
    • 但是当继承的 constructor 执行时，它不会执行此操作。它期望父类的 constructor 来完成这项工作。

    因此，派生的 constructor 必须调用 super 才能执行其父类（base）的 constructor，否则 this 指向的那个对象将不会被创建。并且我们会收到一个报错。

[[HomeObject]]

为了提供解决方法，JavaScript 为函数添加了一个特殊的内部属性：[[HomeObject]]。

  • 方法在内部的 [[HomeObject]] 属性中记住了它们的类/对象。这就是 super 如何解析父方法的。
  • 因此，将一个带有 super 的方法从一个对象复制到另一个对象是不安全的。

普通对象的例子：

正如我们之前所知道的，函数通常都是“自由”的，并没有绑定到 JavaScript 中的对象。正因如此，它们可以在对象之间复制，并用另外一个 this 调用它。

[[HomeObject]] 的存在违反了这个原则，因为方法记住了它们的对象。[[HomeObject]] 不能被更改，所以这个绑定是永久的。

在 JavaScript 语言中 [[HomeObject]] 仅被用于 super。所以，如果一个方法不使用 super，那么我们仍然可以视它为自由的并且可在对象之间复制。但是用了 super 再这样做可能就会出错。

[[HomeObject]] 是为类和普通对象中的方法定义的。但是对于对象而言，方法必须确切指定为 method()，而不是 "method: function()"。

这个差别对我们来说可能不重要，但是对 JavaScript 来说却非常重要。
